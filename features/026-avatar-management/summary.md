# Feature 026 — Avatar Management

**Initial prompt:**

Follow instructions in plan.prompt.md.
As a user, in the profile page, I would like to be able to change my avatar. In the input, you should see a preview of the current avatar if it exists. You can only have one avatar.

In the HomePage, if the user has an avatar, you will display the avatar instead of the first letter of his name.

When starting the implementation, you will create the documentation based on features/ARCHITECTURE.md.

Do you have any questions ?

---

## User Stories

| User Story                                                                                                                                    | Status    |
| :-------------------------------------------------------------------------------------------------------------------------------------------- | :-------- |
| As a user, in the profile page, when I click "Changer mon avatar", I should see a file picker to select an image from my device               | planned   |
| As a user, in the profile page, when I select an image, I should see an immediate preview of the selected image                               | planned   |
| As a user, in the profile page, when I have an existing avatar, I should see it as the current preview before selecting a new one             | planned   |
| As a user, in the profile page, when I confirm the avatar upload, the previous avatar is deleted and replaced by the new one                  | planned   |
| As a user, in the BottomNav, when I have set an avatar, I should see my avatar photo instead of the generic user icon                         | planned   |
| As a user, on the HomePage seller cards, when a seller has an avatar, I should see their photo instead of their initial (already implemented) | completed |
